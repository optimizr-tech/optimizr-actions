# Security policy

## Supported scope

Report vulnerabilities in the workflows, composite actions, scripts, and
documentation in this repository.

## Reporting a vulnerability

Use GitHub's private security advisory flow for this repository. Do not report
security issues in a public issue when the report contains a secret, host or
network detail, customer data, exploit payload, or production configuration.

Include the affected path and revision, impact, safe reproduction steps, and
any proposed mitigation. Do not include live credentials or personal data.

## Handling rules

Maintainers will validate the report privately, prepare a remediation, and
coordinate disclosure after affected consumers have had a reasonable update
window. Fixes must include a regression test whenever the issue has a
testable behavior.
