# Public Actions and Fiscal Contract Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep reusable, secret-free organization automation public while delivering the portable local validation contract and safely restoring the `v1` release gate.

**Architecture:** The repository exposes generic workflows, composite actions and a standard-library local validation runner. Product repositories own secrets, Compose lifecycle and product-specific RLS assertions; the Fiscal preset declares those checks mandatory without embedding credentials or infrastructure details. A static Python suite guards public-boundary policy and the actionlint regressions.

**Tech Stack:** GitHub Actions YAML, Bash, Python 3.12+ standard library, `unittest`.

## Global Constraints

- No credentials, host addresses, customer data or product configuration in the public repository.
- Public workflows must remain backward compatible for existing `@v1` consumers.
- New validation logic uses the Python standard library only.
- Required and unknown validation results fail closed.
- `v1` moves only after repository validation succeeds.
- `optimizr-infra-ops` is private legacy context only; no private path, host, secret, or private-repository dependency is introduced here.
